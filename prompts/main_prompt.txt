3. FM130 Parameter List (PDF):
   - These parameters are used to configure the FM130 device and change the device behaviors(sending location, warns etc).
   - Includes a list of parameter IDs for the FM130 device.
   - These parameters are used via `setparam` command.
   - Parameter IDs:  are used to set or get specific device settings.
   - Parameter Type: Uint8, Uint16, etc.
   - Default value: Varies by parameter.
   - Value range(Min ,Max): Defines the acceptable range for the parameter value.
   - Value: The current value of the Parameter name.
   - Parameter name: Descriptive name of the parameter.
   - Parameters
        -Priority:
        0 - Disabled
        1 - Low (just device warns driver)
        2 - High (both device warns and send mail)
        3 - Panic (send SMS and call and device warns)
        - Operand
        0 - On Exit (when device exits the area)
        1 - On Entrance (when device enters the area)
        2 - On Both (when device enters or exits the area)
        3 - Monitoring (when device is in the area)
        4 - On Hysteresis (when device enters or exits the area with hysteresis)
        5 - On Change (when device enters or exits the area with change)
        6 - On Delta Change
        - Event only
        0 - Disabled
        1 - Enabled

   - Example 1:
     - System parameter: ID = 50180
     - SMS format: `<SMS login> <SMS password> setparam 50180:1`
     - 1 is value and and it means " low priority". Priority table is in Parameters on PDF.
     - Parameter ID ,Parameter Type, Default value, Value range(Min ,Max), Value, Parameter name
     - 50180, Uint8, 0, 0, 3, Analog Input 1 Priority
     - That means Analog Input 1 Priority is set to "low priority" with value 1.
    - Example 2:
     - System parameter: ID = 45100
     - SMS format: `<SMS login> <SMS password> getparams 50180`
     - returns the current value of the parameter name. Example: "Analog Input 1 Priority is set to low priority with value 1.". It means that the current value of Analog Input 1 Priority is 1 so "Low Priority".

Teltonika FMB120 — Veri Gönderim Parametreleri
=============================================

Bu doküman, Teltonika FMB120 GPS cihazında araç hareketliyken veya durağanken ne zaman ve hangi koşullarda veri gönderileceğini belirleyen parametreleri açıklar. Ayarlar SMS ile, FMB Configurator ile veya FOTA üzerinden yapılabilir.

Amaçlar:
- Araç durağan haldeyken 10 dakikada bir konum gönderimi
- Araç hareketliyken 30 saniyede bir konum gönderimi
- Home, Roaming ve Unknown GSM durumlarına göre ayrı ayrı yapılandırma

--------------------------------------------------
1. STOP MODE (DURAĞAN HAL)
--------------------------------------------------

Amaç: Araç hareketsizken 600 saniyede (10 dakika) bir kayıt ve gönderim

Parametreler:
- 10000 = 600         // recording.timeout.stop
- 10004 = 1           // recording.records.min.stop
- 10005 = 1           // recording.panic.record.stop

SMS komutu:
setparam 10000:600;10004:1;10005:1

--------------------------------------------------
3. HOME NETWORK MOVE MODE (EVENT BASED)
--------------------------------------------------

Amaç: Home GSM ağına bağlıyken hareketli modda sadece süreye dayalı olarak veri gönderimi (30 saniye)

Parametreler:
- 10050 = 30          // Min Period (saniye)
- 10051 = 0           // Min Distance (metre)
- 10052 = 0           // Min Angle (derece)
- 10053 = 0           // Min Speed Delta (km/h)
- 10054 = 1           // Min Saved Records
- 10055 = 30          // Send Period (saniye)

SMS komutu:
setparam 10050:30;10051:0;10052:0;10053:0;10054:1;10055:30


--------------------------------------------------
4. ROAMING NETWORK STOP MODE
--------------------------------------------------

Amaç: Roaming durumunda araç durağanken 600 saniyede bir veri gönderimi

Parametreler:
- 10100 = 600         // Min Period
- 10104 = 1           // Min Saved Records
- 10105 = 600         // Send Period

SMS komutu:
setparam 10100:600;10104:1;10105:600

--------------------------------------------------
4. ROAMING NETWORK MOVE MODE (EVENT BASED)
